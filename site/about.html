About give-me-job

give-me-job is a hub-and-spoke collective of specialized Claude Code agents, commands, skills, and Python scripts that, given a real job offer, autonomously produces a truthful, offer-optimized set of application artifacts — a CV (PDF), a cover letter, and an interview-prep document — for one candidate.

Core value

Given a real offer, the system produces application artifacts that provably trace back to the candidate's profile (config/candidate.yaml) and pass mandatory quality gates. If everything else fails, the artifacts must never fabricate and must actually target the offer. Reframing and emphasis are allowed; invention is hard-blocked.

Hub-and-spoke model

A single top-level orchestrator, gmj-orchestrator, is the only role that holds Task and delegates to spokes. Spokes never spawn spokes (nested hubs lose Task in Claude Code), so routing stays a clean User Request → Routing → Agent Selection → Task Delegation → Quality Gate → Result loop. This topology preserves criteria tracking and prevents chain drift.

The collective is exactly nine members: the hub, five core spokes (gmj-offer-scout, gmj-artifact-composer, gmj-truth-verifier, gmj-fit-evaluator, gmj-cv-generator), two retained supporting agents (gmj-candidate-analyzer, gmj-candidate-configurator), and an optional branded-template spoke (gmj-template-creator).

Truthfulness guarantee

config/candidate.yaml is the single source of truth. Every artifact claim must trace to it. Two hard gates — Gate A (truth) and Gate B (target-fit) — are non-bypassable in any mode. Autonomous mode removes the human pause, never the machine gate; auto-loops are bounded by a retry cap.
