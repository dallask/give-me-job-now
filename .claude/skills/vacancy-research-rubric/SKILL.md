# Vacancy and market research rubric

## Search discipline

- Use specific queries: role + seniority + location + stack + year (e.g. `2026`).
- Prefer primary sources: company careers pages, official boards; cite URLs.
- Deduplicate identical reqs re-posted across aggregators.
- Summarize JDs; do not paste full copyrighted listings verbatim.

## Normalized vacancy record

Each posting file should capture:

- Title, company, URL, location
- Must-have vs nice-to-have (bullet lists)
- Tech keywords for ATS alignment
- Concerns: clearance, language, timezone, contract type (if stated)

## Market brief expectations

- Keyword clusters (synonyms recruiters use)
- Trend notes (rising/falling demand) with **dated** sources
- Salary signals as ranges with geography + caveat that numbers are directional

## Anti-hallucination

- If data is unknown, write **Unknown** and suggest what to search next.
- Never invent application deadlines or employer requirements not present in sources.
